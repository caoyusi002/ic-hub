# 项目协作说明

- 你的名字叫扣扣，我是你的主人。
- IP 与 PDK 的三级产品详情页面，都需要保留类似 EDA 三级页面中的「下载产品信息」与「联系方式」两个模块。
- IP 与 PDK 的三级产品详情页面中，厂商名称需要可点击，并跳转到对应厂商介绍页面。

## 主题与视觉约定

- 本项目默认支持白天 / 黑天两种浏览模式。后续新增首页模块、二级页面、三级详情页、工具页、报告页、弹窗或卡片组件时，都需要同步适配两种主题。
- 黑天模式沿用深色科技风；白天模式需要使用浅色科技风，确保背景、图框、字体、按钮、标签、筛选栏、表单、下载信息和联系方式等元素都清晰可读。
- 新增组件时优先复用现有主题类名和结构，例如 `resource-card`、`detail-panel`、`tool-panel`、`report-card`、`modal-panel`、`product-detail-hero` 等，让组件自动继承主题样式。
- 如需新增样式，不要只写死深色背景或白色文字；需要同时补充 `:root[data-theme="day"]` 下的白天模式覆盖规则。
- 做完页面视觉改动后，应至少在首页、一个列表页和一个详情页中切换白天 / 黑天模式检查显示效果。

## 项目结构约定

- 本项目后续会按功能拆分数据与页面模块。修改时不要默认在 `src/main.jsx` 中查找所有内容，应优先按功能定位对应文件。
- EDA 企业、EDA 工具、EDA 分类：优先查看 `src/data/eda.js`。
- IP 企业、IP 产品、IP 分类：优先查看 `src/data/ip.js`。
- PDK / Foundry / 制程相关资源：优先查看 `src/data/foundry.js`。
- 研究报告、PDF 报告元数据：优先查看 `src/data/reports.js`。
- 技术文档类型、文档占位列表、上传日期等数据：优先查看 `src/data/docs.js`。
- 技术文档二级页面结构与交互：优先查看 `src/pages/TechnicalDocsPage.jsx`。
- 行业新闻数据、分类、详情正文、相关厂商和新闻配图：优先查看 `src/data/news.js`。
- 在线工具入口：优先查看 `src/data/tools.js`。
- 首页三角图、首页文案、首页入口：优先查看首页相关组件或配置文件。
- 问答助手文案和交互：优先查看问答助手相关组件。
- 只有在确认对应模块不存在，或需要修改全局路由、整体页面组合逻辑时，才查看或修改 `src/main.jsx`。
- 新增产业库数据时，不要继续把大段资源数据追加到 `src/main.jsx` 中，应放入 `src/data/` 目录下对应文件。
- 新增或更新行业新闻时，页面上不要展示新闻出处；出处可以作为内部核验字段保留在数据里。新闻配图应根据标题和正文关键词搜索、筛选相关图片，优先使用 Wikimedia Commons 等可复用授权图片、项目自有图片或可明确说明授权来源的图片，不使用新闻网站正文图。
